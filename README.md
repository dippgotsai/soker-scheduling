# 門市排班系統

供**百貨專櫃**與**街邊門市**共用的排班系統，內建**台灣勞基法**即時檢核（支援**八週變形工時**與標準工時一例一休）。

## 功能總覽

### 前台（員工）
- 查看個人／門市月班表、未來班次
- 請假申請（特休、補休、事假、病假、生理假、婚喪假、產假、陪產假、家庭照顧假⋯）
- 加班申請（選擇加班費或 1:1 換補休）
- 換班申請（與同事互換或單向轉讓；對方同意 → 店長核准 → 自動改班表並重新法遵檢核）
- 劃休（於開放期間提交希望休假日，店長排班時可見）
- 特休／補休餘額查詢（含補休到期提醒）

### 後台（主管）
- 月班表排班（即時勞基法檢核：違規「錯誤」預設阻擋，可強制排入；「注意」放行並提示）
- 例假／休息日標記（供七休一、八週變形 16 日休假檢核）
- 審核中心（請假／加班／換班簽核）
- 人力需求設定（各星期日 × 班別最少人數，排班頁顯示人力缺口）
- 班別自訂（名稱、起訖、休息時間、顏色；跨日班支援；實際工時 >12 小時禁止建立）
- 劃休開放期間與每人上限設定
- 假日設定（國定假日／門市公休）

### 系統管理（admin）
- 門市管理：百貨／街邊型態、營業時間、**工時制度（標準／八週變形）**、八週週期起算日、每月加班上限（46／54 小時）、內規（最多連上天數、禁止晚接早）
- 帳號管理：角色（管理員／區域主管／店長／員工）、所屬門市（可跨店）、妊娠／未成年標記

## 勞基法檢核規則

| 規則 | 條文 | 系統行為 |
|---|---|---|
| 每日正常＋延長 ≤ 12 小時 | §32 | 錯誤（阻擋） |
| 排班 > 8 小時提示屬延長工時 | §24/§32 | 注意 |
| 班次間隔 ≥ 11 小時 | §34 | 錯誤 |
| 每 7 日至少 1 例假（連 6 上限；八週變形例假挪移上限連 12） | §36 | 錯誤 |
| 標準工時：每週正常 ≤ 40、每 7 日 1 例假＋1 休息日 | §30/§36 | 錯誤 |
| 八週變形：每日 ≤ 8、每週 ≤ 48、8 週 ≤ 320 小時、8 週休假 ≥ 16 日 | §30-1 | 錯誤（未排完週期為注意） |
| 每月加班 ≤ 46 小時（可設 54，另提醒 3 個月 138 上限） | §32 | 錯誤／注意 |
| 未成年工 20:00–06:00 禁排班、每日 ≤ 8 小時 | §47/§48 | 錯誤 |
| 妊娠／哺乳 22:00–06:00 禁排班 | §49 | 錯誤 |
| 特休依年資自動核給（週年制：3/7/10/14/15/…30 日） | §38 | 自動 |
| 加班換補休 1:1，補休期限屆期提醒折發工資 | §32-1 | 自動 |
| 加班費率試算（平日 1.34/1.67；休息日 1.34/1.67/2.67；國定假日加倍） | §24/§39 | 顯示 |

> 注意：採八週變形工時之門市，須為勞動部指定行業（零售業屬之）**並經工會或勞資會議同意**後方可實施；系統於門市設定頁提供開關，程序請自行完成。出勤紀錄（打卡）不在本系統範圍。

## 技術架構

- Next.js 15（App Router、Server Actions）＋ TypeScript ＋ React 19
- SQLite（better-sqlite3），首次啟動自動建表；正式環境可以 `DATABASE_PATH` 指定路徑，之後可平移 PostgreSQL
- 自建 Session 認證（bcrypt 密碼雜湊、HttpOnly Cookie）、RBAC 權限
- 無前端框架依賴的 RWD 介面，手機可用

## 快速開始

```bash
npm install
npm run seed     # 建立系統假別與示範資料
npm run dev      # http://localhost:3000
```

## 部署到 Railway

本 repo 已含 `Dockerfile` 與 `railway.json`，首次啟動會自動建表與建立初始資料：

1. 到 [railway.app](https://railway.app) 登入 → **New Project → Deploy from GitHub repo** → 選擇本 repo
2. 部署完成前，到服務的 **Settings → Volumes → Add Volume**，Mount path 填 **`/data`**（SQLite 資料檔位置，沒掛 Volume 重佈署會清空資料）
3. **Settings → Networking → Generate Domain** 產生公開網址
4. 環境變數（選填）：`SEED_DEMO=0` 不建示範資料（只建 A001 管理員）；`ADMIN_PASSWORD=...` 指定管理員初始密碼

部署後用示範帳號登入（見上表），**正式使用前請先更改所有預設密碼**。
